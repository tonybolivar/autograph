# Autograph autofill coverage audit, 2026-05-13

Snapshot of per-ATS field coverage after a Playwright-driven harness pass.

Master profile now has **101 fields** across about, experience, education, employment, logistics, preferences, writing, and demographic sections.

## Global improvements landed in this pass

- Demographic radios fall back to "Prefer Not To Answer" / decline labels when the canonical profile value (race, gender, etc.) does not match any visible option.
- Radio fill now skips if a peer in the same name group is already checked, avoiding double-fill when decline labels and canonical values both match.
- AG_DECLINE_OPTION_LABELS expanded to include "Prefer not to disclose", "Prefer not to say", "Choose Not to Disclose", "I don't wish to disclose", and other common phrasings seen in Lever surveys and similar.
- Pronoun denormalizer added so `personal_pronouns` (e.g. "he/him") maps to the visible "He/him" / "He" / "Him" checkbox labels used by Lever and others.
- labelMatcher tokenization now treats parentheses and square brackets as word separators so labels like "Location (City)" match the city alias set.
- Greenhouse fillDropdown now uses a mousedown sequence and a keyboard fallback to open react-select menus that ignore plain click(), restoring the EEOC Gender/Race/Veteran/Disability dropdowns.

## Iteration 3 fixes (user-driven follow-ups)

After round-2, the user flagged two real defects observed in production:

- TeamTailor was re-uploading the same resume on every mutation observer tick. Root cause: after the first upload, TeamTailor swaps the file input element for a fresh one; the new element is not in the WeakSet, so `fillResumeInputs` picks it up again. Fixed: `shouldFillResumeInput` now returns false when `candidate[resume_remote_url]` already holds an `https://` value, so we know the upload completed.
- Recruitee Hispanic-or-Latino radio (and similar Vue v-model radios) had `data-ag-filled=true` but `el.checked` reverted to false. Root cause: Vue listens for the bubbling click on the label wrapper and our `agSetCheckedSafe` only dispatched a plain `click()`. Fixed: Recruitee adapter now dispatches a `mousedown`+`mouseup`+`click` sequence on the radio's sibling/associated label. Confirmed: Hispanic or Latino, state of residence (New York), Male, sponsorship-yes all now check correctly.
- Eightfold privacy modal loop: user reported the Data Privacy modal re-appearing after every fill. Defaults investigation showed "Save my answers for future applications" is pre-checked by the page (defaultChecked: true, agFilled: false), so Autograph wasn't responsible for that. Mitigations added regardless: `waitForReady` delays first fill, modal/dialog containers are excluded from fill, consent/privacy/i-agree/acknowledge checkboxes are explicitly excluded, and `suppressRefillOnRerender: true` prevents re-fills during the page's React re-renders.
